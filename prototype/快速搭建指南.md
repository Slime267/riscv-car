## 分步接线

### 步骤 1: 供电 (先接这个, 用万用表确认)

```
电池+ (红) ──→ MP1584 IN+
电池- (黑) ──→ MP1584 IN-
电池+ (红) ──→ L298N 12V 排针
电池- (黑) ──→ L298N GND 排针 (与 MP1584 GND 共地)

MP1584 OUT+ ──→ ESP32C3 5V 引脚
MP1584 OUT- ──→ ESP32C3 GND 引脚
MP1584 OUT+ ──→ L298N 5V 排针  ← 拔掉跳线帽!

ESP32C3 3.3V ──→ SSD1306 VCC
ESP32C3 GND  ──→ SSD1306 GND
```

> 上电前用万用表确认 MP1584 输出真的 5.0V。确认 SSD1306 接的是 3.3V 不是 5V。

### 步骤 2: 信号线

```
ESP32C3      L298N
─────────────────────
GPIO2   ──→  ENA
GPIO3   ──→  IN1
GPIO4   ──→  IN2
GPIO5   ──→  ENB
GPIO10  ──→  IN3
GPIO1   ──→  IN4
GND     ──→  GND (已有的共地)

ESP32C3      SSD1306
─────────────────────
GPIO6   ──→  SDA
GPIO7   ──→  SCL
3.3V    ──→  VCC  (已经接了)
GND     ──→  GND  (已经接了)
```

### 步骤 3: 电机

```
L298N OUT1 ──→ 左电机 红线
L298N OUT2 ──→ 左电机 黑线
L298N OUT3 ──→ 右电机 红线
L298N OUT4 ──→ 右电机 黑线
```

每个电机两端并联一颗 104 (0.1μF) 电容消火花。

### 步骤 4: 电池分压 (可选但建议接)

```
电池+ ── [10KΩ] ──┬── GPIO0
                  ├── [20KΩ] ── GND
电池- ─────────────── GND
```

## 烧录

### 1. Arduino IDE 配置

```
附加开发板网址:
  https://espressif.github.io/arduino-esp32/package_esp32_index.json

库管理 安装:
  Adafruit SSD1306
  Adafruit GFX
  Adafruit BusIO

开发板:
  ESP32C3 Dev Module
  USB CDC On Boot: Enabled
  CPU: 160MHz
  Flash: 4MB
```

### 2. 烧录

USB 数据线连接电脑 → 点 Upload → 出现 "Connecting..." 时按住 BOOT 键 → "Writing..." 时松手。

### 3. 上电测试

拔 USB → 开电池开关 → OLED 显示 "RISC-V" → 3 秒后进入主界面 → 显示 "BLE:--"

手机 Chrome 打开 `web/index.html` → 点"连接小车" → 选 "RISC-V Car" → OLED 变 "BLE:OK" → 方向键遥控。

---

## 方向纠正

如果按前进键车子却往后退 → 把两个电机的红黑线对调。
如果按左转车子右转 → 把左右电机的 OUT 端子对调。
如果按前进车子打转 → 两个电机转向不一致，对调其中一个的红黑线。

## 故障排查

| 现象 | 检查 |
|------|------|
| 上电 OLED 不亮 | SSD1306 VCC 必须接 3.3V, SDA/SCL 线序 |
| L298N 不工作 | 5V 跳线帽拔了没? ENA/ENB 跳线帽插了没? |
| 电机不动 | 用万用表测 L298N OUT 端有电压吗? 电池有电吗? |
| 电机一转 BLE 断连 | 104 电容没焊, 电机干扰太大 |
| 无法烧录 | USB 线是数据线吗? 按住 BOOT 键了吗? |
| BLE 搜不到 | 串口显示 "BLE 广播中" 了吗? Android 要开位置服务 |
